# Web版知识库问答系统 - 业务需求文档

## 1. 项目背景

### 1.1 现有系统
- **项目名称**：rag-skill
- **核心能力**：基于本地知识库的智能检索问答系统
- **技术特点**：
  - 无需向量数据库
  - 基于 `data_structure.md` 分层索引导航
  - 渐进式检索（grep + 局部读取）
  - 支持多种文件格式（Markdown、PDF、Excel）
  - 多轮迭代检索（最多5次）

### 1.2 现有局限
- 仅支持本地 IDE 环境
- 单用户使用
- 无法通过浏览器访问
- 缺少用户管理和会话持久化

## 2. 业务目标

### 2.1 核心目标
将 rag-skill 从本地 IDE 环境迁移到 Web 平台，实现：
- 多用户并发访问
- 浏览器端对话体验
- 思考过程实时展示
- 信息来源追溯
- 模型灵活切换

### 2.2 使用场景
- **部署环境**：公司内网
- **用户规模**：企业内部用户（预计 10-100 人）
- **使用方式**：浏览器访问，无需安装
- **知识库类型**：多领域、多格式（PDF、Excel、Markdown）

## 3. 功能需求

### 3.1 核心功能

#### 3.1.1 知识库检索
- **分层索引导航**
  - 自动读取 `data_structure.md` 了解知识库结构
  - 递归定位相关目录和文件
  - 支持多级目录索引

- **文件类型支持**
  - Markdown/文本：直接读取
  - PDF：解析提取文本（PyPDF2）
  - Excel：数据查询分析（pandas）
  - **接口预留**：未来支持 HTML、Word 等格式

- **渐进式检索**
  - 使用 grep 定位关键词
  - 局部读取匹配内容（避免全文件加载）
  - 多轮迭代（最多5次）
  - 结果重排序和去重

#### 3.1.2 智能问答
- **问题理解**
  - 提取关键词、主题
  - 识别时间范围、输出类型
  - 理解用户意图

- **答案生成**
  - 基于检索结果生成答案
  - 标注信息来源（文件路径、行号）
  - 支持流式输出

#### 3.1.3 思考过程展示
- **工具调用链展示**
  - 实时显示正在执行的工具
  - 显示工具参数和结果预览
  - 显示检索迭代过程

- **WebSocket 实时推送**
  - 连接建立通知
  - 思考步骤推送
  - 答案流式推送
  - 来源信息推送

#### 3.1.4 来源追溯
- **来源标注**
  - 文件路径
  - 行号/页码
  - 相关性评分

- **来源交互**
  - 点击跳转到原始文件
  - 预览匹配内容
  - 下载原始文件

### 3.2 用户管理

#### 3.2.1 会话管理
- **匿名会话**
  - 基于 IP 识别
  - 无需登录
  - 会话数据存储在浏览器 localStorage

- **会话数据**
  - 会话 ID
  - 用户偏好（主题、语言、字体大小）
  - 对话历史（可选，IndexedDB）

#### 3.2.2 用户偏好
- **支持设置**
  - 界面主题（亮色/暗色）
  - 系统语言（中文/英文/日文）：界面显示语言
  - 大模型输出语言（中文/英文/日文）：AI 回答语言，默认中文
  - 字体大小
  - 思考过程显示开关

- **存储位置**
  - localStorage（轻量数据）
  - IndexedDB（对话历史）

### 3.3 模型管理

#### 3.3.1 模型配置
- **默认模型**：Qwen2.5-7B-Instruct
- **次选模型**：Yi-1.5-9B-Chat
- **备选模型**：Llama-3.1-8B-Instruct

#### 3.3.2 模型切换
- **配置文件**：`config/models.yaml`
- **切换方式**
  - API 接口切换
  - 前端设置页面
  - 实时生效

- **模型信息展示**
  - 模型名称
  - 参数量
  - 内存占用
  - 支持语言
  - 能力描述

### 3.4 多语言支持

#### 3.4.1 输入语言
- **支持语言**：中文、英文、日文
- **自动识别**：根据用户提问语言

#### 3.4.2 输出语言
- **用户设置**：UI 层面独立设置大模型输出语言
- **支持选项**：中文、英文、日文
- **默认行为**：默认中文，不跟随输入语言
- **独立于系统语言**：与界面显示语言完全分离

#### 3.4.3 知识库语言支持
- **支持语言**：中文、英文、日文
- **跨语言检索**：用户用中文提问，可检索日语知识库内容
- **多语言适配**：在问题识别、知识库匹配、答案生成阶段均做好多语言适配
- **语言转换**：自动处理不同语言之间的转换，确保语义准确

### 3.5 知识库管理

#### 3.5.1 文件上传
- **支持格式**：PDF、Excel、Markdown、TXT
- **上传方式**：拖拽上传、点击上传
- **批量上传**：支持多文件同时上传
- **智能解析**：通过大模型自动解析文件内容，提取关键信息
- **接口预留**：未来支持 HTML、Word 等格式

#### 3.5.2 知识库结构
- **自动索引**
  - 上传后自动解析
  - 生成 `data_structure.md`
  - 更新索引树

- **目录管理**
  - 创建/删除目录
  - 移动文件
  - 重命名文件

#### 3.5.3 实时更新
- **文件监听**
  - 监控知识库目录变化
  - 自动更新索引
  - 无需手动刷新

## 4. 非功能需求

### 4.1 性能需求
- **响应时间**
  - 首次响应 < 2 秒
  - 流式输出延迟 < 500ms
  - 检索时间 < 5 秒

- **并发支持**
  - 支持 10+ 并发用户
  - 单用户多会话支持

### 4.2 可用性需求
- **浏览器支持**
  - Chrome 90+
  - Firefox 88+
  - Edge 90+
  - Safari 14+

- **设备支持**
  - PC 端优先
  - 平板适配
  - 移动端适配（可选）

### 4.3 可靠性需求
- **错误处理**
  - 友好的错误提示
  - 自动重试机制
  - 日志记录

### 4.4 可维护性需求
- **日志记录**

  - 资源使用监控

## 5. 约束条件

### 5.1 技术约束
- **部署环境**：公司内网
- **内存限制**：本地模型内存占用 < 6GB
- **无外部依赖**：不依赖 Redis、PostgreSQL、向量数据库

### 5.2 业务约束
- **知识库格式**：必须包含 `data_structure.md` 索引文件
- **模型能力**：必须支持工具调用（Function Calling）
- **多语言**：必须支持中英日三语

### 5.3 安全约束
- **数据不出内网**：所有数据在内网流转
- **用户隐私**：会话数据本地存储
- **访问控制**：IP 限流、会话隔离
