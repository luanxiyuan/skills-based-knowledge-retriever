# Web版知识库检索工具 - 架构设计文档

## 1. 项目概述

### 1.1 项目背景
本项目旨在构建一个Web版知识库检索工具，采用类似Trae执行Skill的方式，无需RAG和向量数据库，通过分层索引和渐进式检索实现高效的多格式文件问答系统。

### 1.2 核心目标
- 实现类似Trae的Skill执行机制
- 支持多格式文件检索（Markdown、PDF、Excel）
- 提供实时思考过程展示
- 支持多用户并发访问
- 实现信息来源追溯

### 1.3 技术特点
- 无向量数据库依赖，基于文件系统和分层索引
- 渐进式检索，按需加载资源
- 多轮迭代优化检索结果
- 实时WebSocket推送执行过程
- Agent协调与大模型多轮交互

### 2.2 核心组件说明

#### 2.2.1 前端层
- **对话界面**：用户输入问题，展示系统回答
- **执行过程展示**：实时展示Skill执行步骤和工具调用
- **来源追溯**：显示检索结果的来源文件和位置
- **知识库浏览**：查看知识库结构和文件内容

#### 2.2.2 API层
- **RESTful API**：处理前端请求，如会话管理、模型配置等
- **WebSocket服务**：实时推送执行状态、思考过程和结果

#### 2.2.3 Agent层
- **任务协调器**：管理整个执行流程，协调各组件工作
- **大模型交互**：与大模型进行多轮对话，获取推理结果

#### 2.2.4 Skill执行引擎
- **渐进式执行器**：实现类似Trae的三层加载机制
- **工具选择器**：根据当前任务选择合适的工具

#### 2.2.5 工具层
- **文件处理工具**：处理Markdown、PDF、Excel等文件
- **检索工具**：实现关键词搜索、局部读取等功能

#### 2.2.6 存储层
- **知识库文件**：存储原始文件（PDF、Excel、Markdown等）
- **索引文件**：data_structure.md文件，提供目录结构信息

### 2.3 核心流程

#### 2.3.1 对话启动流程
1. 用户在前端输入问题
2. 前端通过API发送问题到后端
3. Agent层接收问题，开始执行流程

#### 2.3.2 Skill执行流程
1. **元数据加载**：解析SKILL.md文件，获取基本信息
2. **核心指令加载**：加载执行流程，理解用户问题
3. **分层索引导航**：读取data_structure.md，定位相关文件
4. **工具选择与学习**：根据文件类型选择合适的工具
5. **渐进式检索**：按文件类型执行检索策略
6. **多轮迭代**：最多5轮检索，逐步优化结果
7. **答案生成与溯源**：汇总结果，生成最终答案

#### 2.3.3 实时反馈流程
1. **执行步骤推送**：每执行一步，通过WebSocket推送状态
2. **工具调用推送**：推送工具调用的参数和结果
3. **结果流式推送**：答案生成过程实时推送到前端
4. **来源信息推送**：推送检索结果的来源信息

## 3. 技术选型

### 3.1 前端技术
- **框架**：Vite + React 18 + TypeScript
- **UI 组件**：shadcn/ui (基于 Radix UI)
- **样式**：Tailwind CSS v4
- **WebSocket**：原生 WebSocket API
- **图标**：Lucide React
- **构建工具**：Vite 5

### 3.2 后端技术
- **框架**：FastAPI + Uvicorn
- **WebSocket**：FastAPI 内置 WebSocket 支持 (websockets 库)
- **并发处理**：asyncio + async/await
- **配置管理**：PyYAML
- **文件处理**：
  - PDF：pdfplumber + pdfminer.six
  - Excel：pandas + openpyxl + xlrd
  - Markdown：Python 标准库 + markdown 库
  - 文本搜索：grep-ast

### 3.3 大模型
- **默认模型**：qwen2.5:7b-instruct (通过 Ollama)
- **备选模型**：
  - qwen2.5:14b-instruct
  - llama3.1:8b-instruct
- **接口协议**：Ollama API (兼容 OpenAI Chat Completion)
- **重试机制**：最多 2 次自动重试（可配置）

### 3.4 存储
- **知识库文件**：本地文件系统 (backend/knowledge/)
- **Skill 定义**：Markdown 文件 (backend/rag-skill/)
- **索引文件**：data_structure.md (分层索引)
- **配置文件**：config.yaml (集中配置管理)
- **会话数据**：浏览器内存 + WebSocket 连接管理

## 4. 关键功能实现

### 4.1 渐进式检索
- **分层索引**：自动解析data_structure.md
- **关键词提取**：从用户问题中提取关键词
- **局部读取**：按行号范围读取文件，避免全文加载
- **多轮迭代**：最多5轮检索，逐步缩小范围

### 4.2 实时思考过程展示
- **WebSocket推送**：实时推送执行步骤
- **可视化组件**：前端展示执行过程
- **交互反馈**：用户可以看到系统的"思考"过程

### 4.3 多文件类型支持
- **Markdown/文本**：直接grep检索 + 局部读取
- **PDF**：先学习处理方法，再提取文本检索
- **Excel**：先学习处理方法，再用pandas分析

### 4.4 多轮交互
- **Agent协调**：管理与大模型的多轮对话
- **逐步优化**：每轮迭代完善检索策略
- **上下文管理**：维护对话上下文，确保回答连贯性

## 6. 安全与性能

### 6.1 安全措施
- **数据不出内网**：所有数据在内网流转
- **会话隔离**：用户会话数据本地存储
- **访问控制**：IP限流、请求频率限制
- **文件上传验证**：验证文件类型和大小

### 6.2 性能优化
- **缓存策略**：缓存检索结果和文件解析结果
- **异步处理**：使用异步IO处理文件操作
- **并行检索**：对多个文件并行检索
- **流式输出**：实现答案的流式生成和推送

## 9. 结论

本架构设计文档提供了一个完整的Web版知识库检索工具的实现方案，采用类似Trae执行Skill的方式，无需RAG和向量数据库，通过分层索引和渐进式检索实现高效的多格式文件问答系统。该方案具有以下优势：

- **无向量数据库依赖**：部署简单，资源消耗低
- **渐进式执行**：提高检索效率，减少token消耗
- **实时反馈**：增强用户体验，提供透明的执行过程
- **多文件类型支持**：适应不同知识库需求
- **与Trae Skill执行方式兼容**：便于迁移现有Skill

通过本架构设计，可以构建一个功能完整、性能优良的Web版知识库检索工具，为用户提供流畅、高效的知识库检索体验。
